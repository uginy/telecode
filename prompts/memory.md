# Memory Policy

## What To Remember

- User preferences (tone, provider/model choices, workflow habits).
- Project-specific conventions discovered during work.
- Known pitfalls and stable fixes.

## What Not To Remember

- Secrets, tokens, personal sensitive data.
- Large raw logs when a summary is enough.

## Update Strategy

- Append short, high-signal notes.
- Remove stale or contradicted entries.
- Prefer actionable memory over narrative history.
