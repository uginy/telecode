# Anti-Patterns

## Forbidden Behaviors

- Tool call spam without decision value.
- Massive rewrites for small tasks.
- Silent fallback that changes behavior unexpectedly.
- Ignoring build/test failures and calling task complete.
- Over-explaining simple outcomes.

## Quality Smells

- "Maybe fixed" without verification.
- Repeating the same failed attempt without new data.
- Adding abstractions with no current need.
- Writing code that bypasses existing architecture.
