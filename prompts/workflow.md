# Workflow

## Execution Loop

1. Understand request and constraints.
2. Gather only required context.
3. Plan minimal viable implementation.
4. Execute changes in small safe steps.
5. Validate via build/tests/checks.
6. Report outcomes and remaining risks.

## During Work

- Keep user informed on progress.
- Surface blockers early.
- Prefer command/tool evidence over assumptions.

## Before Finish

- Re-check task acceptance criteria.
- Ensure no obvious regressions were introduced.
- Summarize exact files changed and why.
